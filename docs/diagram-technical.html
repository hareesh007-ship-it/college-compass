College Compass — How it works

Plain-English explanation alongside the technical details for each pipeline step

Pipeline at a glance

Step In plain English What's happening technically
1 Read your profile
You fill in an Excel sheet with GPA, test scores, budget, state, and major. Optionally drop in a transcript or resume PDF — the tool reads them and fills any gaps you left blank. load_profile.py reads the Excel. If PDFs or docs exist in input/, extract_student_docs.py runs an LLM 🤖 LLM to extract fields like GPA, courses, and activities into any blank cells. Excel values always win. Skipped entirely if no docs are present.
2 Find matching schools
The tool searches a free government database of thousands of colleges, filtered by your state, budget, and major. Any schools you already listed are always included. Write a sentence about what you're looking for ("strong engineering schools in the Midwest") and it adds suggestions automatically. discover_colleges.py queries the College Scorecard API and saves results to data/colleges/catalog.json. llm_discover.py 🤖 LLM turns the any_other_preference free-text field into school names. Skipped if any_other_preference is blank.
3 Research each school
For schools the tool hasn't seen before, it visits each school's admissions website and pulls out key numbers — test score ranges, acceptance rates, application deadlines. This is saved locally so it never has to do it again. auto_research_colleges.py fetches each college's admissions page and passes it to an LLM 🤖 LLM to extract mid-50% GPA/SAT/ACT bands, acceptance rates, and deadlines into data/college_research_cache.json. Schools already in the cache are skipped entirely. Cache is shared across all students.
4 Classify each school
Pure math — no AI. Your GPA and test scores are compared to each school's admit profile. Schools are sorted into Safety, Target, or Reach based on how your numbers compare to their ranges and acceptance rates. college_finder.py runs deterministic Python rules against cache data. acceptance_data.py resolves accept rates (cache-first). program_admit.py evaluates whether the student meets program-specific requirements. No LLM involved.
5 Generate outputs
Two files land in your output folder: a 27-column Excel spreadsheet and a printable gap analysis report. Open both and review your list. build_selection_sheet.py writes the Excel (rankings, tuition, net price, deadlines, fit). build_gap_analysis.py produces the HTML gap report. Both are pure Python — no LLM involved.

Where LLM is used — 3 places only

① Document extraction — extract_student_docs.py Reads unstructured PDF/DOCX files (transcript, resume) and fills any blank fields in the Excel profile — GPA, courses, activities, awards. Skipped entirely if no docs are present in input/.
② College auto-research — auto_research_colleges.py Fetches each college's admissions page and asks the LLM to extract structured data — mid-50% bands, acceptance rates, deadlines — into the shared cache. Each school is only researched once. Cached schools are never fetched again.
③ School name suggestions — llm_discover.py Turns a free-text preference (e.g. "strong engineering programs in the Midwest") into actual school names to add to the discovery pool. Skipped if any_other_preference is blank.
"Python decides. Models draft. Humans validate." Once the cache is warm and the profile is complete, the entire pipeline runs in pure Python with zero LLM calls.
